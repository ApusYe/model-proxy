### [Assertion 3] Task 8: `TokenStats.add()` method has a double dictionary subscript that may not accumulate correctly -- the two statements `records[vendorID, default: [:]][modelID, default: ModelTokenRecord()].inputTokens += input` and the equivalent for `outputTokens` each independently look up with `default:`, so the second `+=` may operate on a fresh default rather than the record modified by the first `+=`.

**Verification:** Read plan lines 350-352. In Swift, `Dictionary` subscript with `default:` returns an inout reference when used with `+=`. However, the two statements are:
```swift
records[vendorID, default: [:]][modelID, default: ModelTokenRecord()].inputTokens += input
records[vendorID, default: [:]][modelID, default: ModelTokenRecord()].outputTokens += output
```
The first statement creates the nested entry if needed and mutates `inputTokens` in place. The second statement accesses the same nested entry (now existing) and mutates `outputTokens`. This works correctly because the `default:` subscript on the outer dictionary inserts the default on first access, and subsequent accesses find the existing entry.
**Result:** Assertion does not hold. The code is correct. The unit test in Task 10 (`tokenStatsAccumulation`) also validates this behavior.

**Hypothesized failure:** `AppConfig` is `@Observable` and used as an environment value via `.environment(configStore.config)`. In SwiftUI, `@Observable` objects are injected with `.environment()` and retrieved with `@Environment(AppConfig.self)`. However, `AppConfig` is a `class` that is `Codable` but not `@MainActor`. Under Swift 6 strict concurrency, passing `AppConfig` across isolation boundaries may emit warnings or errors.
**Plan coverage:** PARTIALLY COVERED. Plan line 443 mentions: "If the compiler emits a `Sendable` warning for `AppConfig`, annotate the class declaration with `@MainActor`." This is a conditional fix instruction. Since `ConfigStore` is `@MainActor` and `config` is accessed from SwiftUI views (also main actor), this should work. But the "if" conditional makes it reactive rather than proactive. This is acceptable for a scaffold phase.

**Plan coverage:** The plan specifies Swift 6, which supports `nonisolated(unsafe)`. COVERED.
