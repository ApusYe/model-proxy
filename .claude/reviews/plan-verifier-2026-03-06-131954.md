### [Compile Failure 3] `ProxyServer` is `@Observable final class` with `@unchecked Sendable` but not an actor -- `isRunning`, `lastError`, `boundPort` are mutated from the async `start()`/`stop()` methods which run on the cooperative pool, while SwiftUI reads them on MainActor. **Hypothesized failure:** `ProxyServer` mutations in `start()` (line 466: `self.isRunning = true`) happen off-MainActor. SwiftUI reads `proxyServer.isRunning` on MainActor. This is a data race under strict concurrency. The `@unchecked Sendable` suppresses the error but does not fix the race.

**Plan coverage:** FAIL -- Not covered. The plan should either:
- Make `ProxyServer` a `@MainActor` class (then NIO operations must be dispatched off-main explicitly), or
- Use `@MainActor`-isolated property wrappers for the observable state, or
- Make it an actor with nonisolated observable properties (complex).

The simplest fix: annotate `ProxyServer` as `@MainActor`. The NIO bootstrap, HTTPClient, and router are all created inside `start()` and used from async contexts; they don't need MainActor. Only the state properties (`isRunning`, `lastError`, `boundPort`) need MainActor isolation, and since `@Observable` needs them all on the same isolation, `@MainActor` on the class is the cleanest approach.

## Decisions

### [DP-001] URL path construction convention (blocking)

**Context:** baseURL includes `/v1` path segment, and `head.uri` also starts with `/v1`, causing path doubling. The convention must be defined before implementation.
**Options:**
- A: baseURL = scheme + host only (e.g., `"https://api.anthropic.com"`) -- head.uri provides full path. Simple, works for Anthropic-compatible APIs. Requires updating `AppConfig.makeDefault()`.
- B: baseURL = full prefix (e.g., `"https://api.anthropic.com/v1"`), strip `/v1` from head.uri before concatenation -- fragile, assumes all vendors use `/v1`.
- C: baseURL = full prefix, head.uri = path after the vendor prefix (e.g., `/messages`) -- requires the proxy to rewrite paths, violates pure-proxy constraint.
**Recommendation:** A -- simplest, no path rewriting needed, consistent with pure-proxy design.

### [DP-002] ProxyServer isolation strategy (blocking)

**Context:** `ProxyServer` needs observable state (for SwiftUI) and async methods (for NIO). These cross isolation boundaries.
**Options:**
- A: `@MainActor` class -- observable state mutations are MainActor-isolated; NIO operations use `await` naturally. Simple.
- B: Actor with `nonisolated` observable properties -- complex, `@Observable` does not work well with actor isolation.
- C: Keep `@unchecked Sendable`, use `DispatchQueue` for synchronization -- manual synchronization, error-prone.
**Recommendation:** A -- `@MainActor` class is the standard SwiftUI pattern for observable state that also does async work.

### [DP-003] How ProxyServer receives routing config (blocking)

**Context:** `ProxyServer.start()` needs routing data from both `AppConfig` (vendors) and `ClientConfig` (modelMappings), but `AppConfig` is non-Sendable.
**Options:**
- A: Pass pre-built `RoutingSnapshot` (Sendable struct) to `start()` -- caller builds it on MainActor synchronously. Clean separation.
- B: Pass both `AppConfig` and `ClientConfig` to `start()`, build snapshot inside -- still has Sendable issue with AppConfig.
- C: Make `AppConfig` Sendable by adding `@unchecked Sendable` -- masks the real issue; AppConfig is mutable and observed.
**Recommendation:** A -- build `RoutingSnapshot` on MainActor, pass only the Sendable snapshot. This is what the plan's architecture intends but the implementation code does not follow through.
