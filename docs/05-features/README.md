# Feature Documentation

Record expected behavior, key code locations, and boundary conditions for each feature.

## File Template

Each feature gets its own file: `feature-name.md`

```markdown
# Feature Name

## Expected Behavior
What the feature does from the user's perspective.

## Key Files
| File | Responsibility |
|------|---------------|
| `path/File.swift` | Description |

## Boundary Conditions
- Condition 1

## Change History
| Date | Change |
|------|--------|
| YYYY-MM-DD | Initial implementation |
```
